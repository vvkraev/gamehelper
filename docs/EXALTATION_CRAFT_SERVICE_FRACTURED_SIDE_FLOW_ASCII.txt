# ExaltationCraftServiceFracturedSide + OmenActivationService — decision flow (ASCII)
#
# Цель: описать алгоритм крафта через Orb of Exaltation с использованием
#       омeнов как вспомогательного механизма. Реализация — в сервисе
#       ExaltationCraftServiceFracturedSide; документ нужен как "истина" для кода.
#
# Важно:
# - OmenActivationService — НЕ режим крафта (не элемент CraftModeCombo),
#   а вспомогательный сервис, который ExaltationCraftServiceFracturedSide вызывает по необходимости.
# - Лимиты 3 Prefix / 3 Suffix для Rare — по умолчанию.
#   Список исключений (другие лимиты) будет добавлен позже.
#
#
# Обозначения
# -----------
# item         : текущий предмет (ParsedItem/clipboard)
# P_max/S_max  : максимумы (обычно 3/3)
# P_cur/S_cur  : текущее количество префиксов/суффиксов на предмете
# P_need/S_need: что "мы хотим получить" по плану условий (из CraftConditionPlan)
# match_count  : сколько аффиксов (по условию) уже удовлетворено сейчас
# can_add_1    : предмет может вместить ещё 1 аффикс (есть свободный слот нужного типа или вообще слот)
# can_add_2    : предмет может вместить ещё 2 аффикса (суммарно свободно >= 2)
# prefix_only  : все аффиксы в условии — Prefix Modifier
# suffix_only  : все аффиксы в условии — Suffix Modifier
# mixed        : в условии одновременно есть и префиксные, и суффиксные аффиксы
#                (ни prefix_only, ни suffix_only)
#
# Копирование текста предмета в буфер (унификация):
# - по **ячейкам омнов** (сетки Sinistral / Dextral / Greater и т.п.) — везде **Ctrl+C**
#   (с очисткой буфера там, где это заложено в шагах);
# - **Ctrl+Alt+C** — только по **ячейке предмета** (крафт по предмету), для полного описания предмета.
#
# Активация омена в ячейке (визуально):
# - если ячейка **обведена красной рамкой**, омен в этой ячейке считается **Activated**;
# - иначе — **Deactivated** (для записи в GreaterExaltationOmenCellStates и проверок в RefreshGreater…).
#
# "Активировать омен" означает:
# - выбрать первую непустую ячейку из соответствующей области омена
# - убедиться по Ctrl+C (с очисткой буфера), что там правильный Omen по имени
# - убедиться, что он активен (красная рамка), иначе ПКМ по ячейке и перепроверка
# - если омeнов не осталось (все ячейки пустые или не тот омен) — остановить крафт
#
# "Деактивировать омены" означает:
# - выбрать первую непустую ячейку из соответствующей области омена
# - убедиться по Ctrl+C (с очисткой буфера), что там правильный Omen по имени
# - убедиться, что он деактивирован (отсутствует красная рамка), иначе ПКМ по ячейке и перепроверка
#
# Вспомогательная функция: какие области Exaltation участвуют в крафте
# --------------------------------------------------------------------
# Общая для AddTwoAffixes и AddOneAffix. Отвечает на вопрос: какой набор областей
# омнов (exaltation) используется в данном крафте, исходя из условий крафта.
#
# Вход: из плана условий известны признаки prefix_only / suffix_only / mixed
#       (см. шаг 2 основного потока).
#
# Правило:
# - Область Greater exaltation — всегда входит в набор (используем всегда).
# - Если условие крафта prefix_only — в набор добавляется область Sinistral exaltation.
# - Если условие крафта suffix_only — в набор добавляется область Dextral exaltation.
# - Если mixed (ни prefix_only, ни suffix_only) — боковая область по этому правилу
#   не добавляется; в наборе только Greater exaltation.
#
# (Имя функции в коде — на усмотрение реализации; в схеме: resolveCraftExaltationAreas.)
#
#
# Переменные в рамках одного крафта
# -------------------------------
# Живут внутри одного «сеанса» крафта по одной ячейке предмета, чтобы не делать
# лишних вызовов вспомогательных функций и не пересчитывать одно и то же.
#
# Имена ниже — условные для кода (PascalCase); в схеме можно ссылаться кратко.
#
#   CraftExaltationAreas              — набор областей exaltation для этого крафта
#   RemainingSinistralExaltationUses  — сколько единиц оменов Sinistral доступно (сумма стаков)
#   RemainingDextralExaltationUses    — сколько единиц оменов Dextral доступно (сумма стаков)
#   RemainingGreaterExaltationUses    — сколько единиц оменов Greater доступно (сумма стаков)
#   GreaterExaltationOmenCellStates   — поячейковое состояние только области Greater (см. п. 3)
#   RefreshRemainingExaltationOmenStacks — функция пересчёта Remaining* по сеткам (см. ниже)
#   RefreshGreaterExaltationOmenCellStates — заполнение GreaterExaltationOmenCellStates (см. ниже)
#   EnsureInitialExaltationOmenSupply — стартовая проверка омнов после CraftExaltationAreas (см. ниже)
#   PreviousCraftBranch — предыдущая ветка крафта: **A** или **B** (см. п. 4)
#
# 1) CraftExaltationAreas
#    Смысл: какие области exaltation задействованы в этом крафте (результат правила выше).
#    Заполнение: один раз при входе в крафт — в блоке «Init craft» (см. Decision flow),
#                вызовом resolveCraftExaltationAreas из плана условий (prefix_only /
#                suffix_only / mixed). При возврате цикла на 1.1 переменную не пересчитываем
#                без необходимости (план условий тот же).
#    Дальнейшие шаги AddTwoAffixes / AddOneAffix опираются на это значение.
#
# 2) RemainingSinistralExaltationUses, RemainingDextralExaltationUses, RemainingGreaterExaltationUses
#    Смысл: оставшееся количество «использований» оменов каждого типа — сумма **N** из строк
#    «Stack Size: N/M» по всем валидным непустым ячейкам соответствующей области сетки (после Refresh).
#    Заполнение: вызовом RefreshRemainingExaltationOmenStacks (см. ниже), а не по формуле «ячейки×10».
#    Проверки «остаток == 0» и пополнение на старте — только для **используемых** областей
#    (см. EnsureInitialExaltationOmenSupply): нули по Sinistral/Dextral **не** считаются проблемой,
#    если соответствующая область **не** входит в CraftExaltationAreas.
#    После расхода оменов в игре — **гибрид**:
#    - обычно уменьшать Remaining* **локально** на известное число потраченных единиц за шаг;
#    - **полный** RefreshRemainingExaltationOmenStacks — после успешного RefillOmen (см. HandleRemaining…),
#      и при **сомнениях** / рассинхроне с игрой (конкретные триггеры полного Refresh — в коде,
#      например несовпадение ожидаемого остатка после шага с контрольной проверкой одной ячейки).
#
# 3) GreaterExaltationOmenCellStates
#    Смысл: массив длины **ровно** количеству ячеек в настроенной сетке **только** области Greater
#    exaltation. Индекс элемента совпадает с порядком обхода ячеек в **RefreshGreaterExaltationOmenCellStates**
#    (0 … CellCountGreater−1).
#    Каждый элемент — пара значений:
#    - **число оменов в ячейке** — целое (для пустой ячейки: **0**);
#    - **статус активации омена в ячейке**: **Activated** или **Deactivated**.
#    Заполнение: **только** функцией **RefreshGreaterExaltationOmenCellStates** (не входит в
#    RefreshRemainingExaltationOmenStacks). Вызов **без флага** — полный снимок всех ячеек Greater;
#    вызов с флагом **Activate** / **Deactivate** — см. раздел RefreshGreater… и начало AddTwoAffixes /
#    AddOneAffix. Статус в ячейке: **красная рамка** = Activated (см. «Активация омена в ячейке» выше).
#
# 4) PreviousCraftBranch (предыдущая ветка крафта)
#    Смысл: какая ветка основного потока последней применила **Orb of Exaltation** к предмету:
#    **A** — ветка **AddTwoAffixes** (match_count == 0, can_add_2); **B** — ветка **AddOneAffix**
#    (match_count > 0, can_add_1). Допустимые значения литералы **A** и **B**.
#    Инициализация: при входе в крафт (блок Init craft) **PreviousCraftBranch := A**.
#    Обновление: **сразу после** успешного применения Orb of Exaltation внутри **AddTwoAffixes**
#    присвоить **A**; внутри **AddOneAffix** — присвоить **B**.
#
#
# RefreshRemainingExaltationOmenStacks — пересчёт остатков омнов по областям сетки
# ---------------------------------------------------------------------------
# Имя в схеме и для кода: RefreshRemainingExaltationOmenStacks
#
# Для каждого типа области (Sinistral / Dextral / Greater), настроенной в конфиге, обходим ячейки
# (на старте и при полном Refresh можно ограничить только теми областями, что в CraftExaltationAreas,
# если нужно сэкономить клики; иначе пересчитать все три, а «нулём» для логики пополнения считать
# только используемые — см. EnsureInitial… и HandleRemaining…).
#
# Для каждой такой области:
# - обойти все ячейки этой области;
# - на каждой ячейке выполнить **Ctrl+C** (текст омена в буфер обмена; см. правило «Копирование…» выше);
# - проверить по тексту, что в ячейке лежит **ожидаемый омен** по полному имени
#   (например «Omen of Sinistral Exaltation» для области Sinistral);
# - из строки вида «Stack Size: N/M» взять **N** (сколько единиц в стаке сейчас; M — вместимость ячейки);
# - просуммировать текущие значения **N** по всем валидным непустым ячейкам области — результат записать
#   в соответствующую переменную Remaining*ExaltationUses.
#
# Пустая ячейка: **не даёт вклада** в сумму, переходим к следующей ячейке.
# Непустая ячейка, но **неверное имя омена** (не совпадает с ожидаемым для этой области):
#   **остановить крафт**, выдать ошибку с двумя полями: что **фактически** в ячейке (текст/имя из буфера),
#   что **ожидалось** (ожидаемое полное имя омена для области).
#
# Пример фрагмента буфера после Ctrl+C по ячейке омена (10 из 10, Sinistral):
# -----------------------------------------------------------------------
# Item Class: Omen
# Rarity: Currency
# Omen of Sinistral Exaltation
# --------
# Stack Size: 10/10
# --------
# While this item is active in your inventory your next
# Exalted Orb will add only prefix modifiers
# --------
# Right click this item in your inventory to set it to be active. This item is consumed when triggered.
# Shift click to unstack.
# -----------------------------------------------------------------------
# Здесь в сумму для Sinistral идёт **10** (число до «/»).
#
#
# RefreshGreaterExaltationOmenCellStates — снимок / корректировка ячеек Greater exaltation
# ---------------------------------------------------------------------------
# Имя в схеме и для кода: RefreshGreaterExaltationOmenCellStates
# Опциональный аргумент (флаг режима): **нет** (полный снимок) | **Activate** | **Deactivate**.
#
# Общее:
# - сетка — только область **Greater exaltation**, индексы i = 0 … CellCountGreater−1;
# - **Activated** / **Deactivated** для ячейки: **красная рамка** вокруг ячейки = **Activated**, иначе **Deactivated**
#   (см. блок «Активация омена в ячейке» в Обозначениях);
# - в сумму **RemainingGreaterExaltationUses** эта функция **не** пишет (остаток Greater — только
#   **RefreshRemainingExaltationOmenStacks**);
# - неверное имя омена (ожидался Greater exaltation omen): **остановить крафт**, ошибка, как в RefreshRemaining….
#
# --- Режим **без флага** (текущая базовая логика) ---
# Обойти **все** ячейки i подряд. На каждой:
# - **Ctrl+C**;
# - если пусто — **GreaterExaltationOmenCellStates[i] := (0, Deactivated)** (или соглашение в коде);
# - иначе проверить имя Greater omen, взять **N** из «Stack Size: N/M», статус по **красной рамке**;
# - **GreaterExaltationOmenCellStates[i] := (N, Activated|Deactivated)**.
#
# --- Режим **Activate** (вызов из **AddTwoAffixes**, если **PreviousCraftBranch == B**) ---
# Взять снимок **GreaterExaltationOmenCellStates на входе вызова** (состояние после **предыдущего**
# обновления массива этой или другой веткой функции). Обойти только индексы i, где **на входе**
# ячейка **непустая** (число оменов в записи > 0). Для каждого такого i:
# - **Ctrl+C**;
# - если по результату ячейка **стала пустой** относительно того, что было до этого прохода
#   (пустой буфер / нет предмета) — записать ячейку **пустой**: **(0, Deactivated)** (соглашение в коде);
# - иначе, если ячейка **не пустая** и по **входному** снимку статус был **Deactivated**:
#     **ПКМ** по соответствующей ячейке на экране → снова **Ctrl+C** (и при необходимости проверка рамки) →
#     обновить **GreaterExaltationOmenCellStates[i]**: актуальное **N** из «Stack Size», статус **Activated**
#     (подтвердить по **красной рамке** после ПКМ).
# - если не пустая и уже **Activated** на входе — достаточно при необходимости обновить **N** из буфера
#   без ПКМ (на усмотрение кода), либо пропустить — зафиксировать в реализации.
#
# --- Режим **Deactivate** (вызов из **AddOneAffix**, если **PreviousCraftBranch == A**) ---
# Аналогично режиму Activate, но обход только **непустых** на **входе** ячеек. Для каждого i:
# - **Ctrl+C**;
# - если ячейка **стала пустой** — записать **пустой** **(0, Deactivated)**;
# - иначе, если **не пустая** и по **входному** снимку статус был **Activated**:
#     **ПКМ** → **Ctrl+C** → обновить **GreaterExaltationOmenCellStates[i]**: **N** и статус **Deactivated**
#     (подтвердить по **отсутствию красной рамки**).
# - если не пустая и уже **Deactivated** на входе — обновление только **N** или пропуск — в коде.
#
#
# EnsureInitialExaltationOmenSupply — старт: омны только в используемых областях
# ---------------------------------------------------------------------------
# Выполняется **после** того, как задан **CraftExaltationAreas** (известно, какие области участвуют
# в крафте), **до** остальной логики Init craft (см. блок Init в Decision flow), чтобы **не**
# срабатывать на RemainingSinistral/RemainingDextral == 0, если эта сторона в крафте не используется.
#
# Greater по правилу resolveCraftExaltationAreas **всегда** в наборе — RemainingGreater всегда
# участвует в проверке.
#
# Примечание реализации: **ЛКМ по _ritualInventoryRegion** выполняется **только** в блоке **RefillOmen**
# (его шаг 1). EnsureInitial **не** делает отдельного ЛКМ по ritual и **не** выполняет контрольного
# пересчёта Remaining* после RefillOmen — остатки берутся из последнего Refresh внутри **RefillOmen** (его шаг 3).
#
# 1) Remaining* ← RefreshRemainingExaltationOmenStacks (пересчёт по нужным сеткам; см. примечание выше).
#
# 2) Если для **любой используемой** области соответствующий счётчик **== 0**:
#      - используемая Sinistral ⇔ область Sinistral входит в CraftExaltationAreas;
#      - используемая Dextral ⇔ область Dextral входит в CraftExaltationAreas;
#      - Greater всегда используется ⇔ RemainingGreaterExaltationUses == 0 попадает в условие.
#    то **немедленно** вызвать **RefillOmen** (как в середине крафта; при необходимости перед этим
#    допускается тот же препроцесс, что в HandleRemainingExaltationUsesDepletion — на усмотрение кода).
#
# 3) Если для **любой используемой** области **Remaining* == 0** (после шага 1 без вызова RefillOmen,
#    либо сразу после завершения **RefillOmen** из шага 2 — без дополнительного Refresh в EnsureInitial):
#      **остановить крафт**, ошибка: **невозможно пополнить область омнами** (в сообщении указать,
#      какая область / какой счётчик нулевой).
#
#
# Вспомогательные сервисы (схема, не отдельные классы обязательно)
# ----------------------------------------------------------------
# Ниже — логические блоки, которые в основном потоке показываются одним вызовом.
# Внутренняя логика задаётся заново.
#
#   HandleRemainingExaltationUsesDepletion (общий флоу: stash-clear + RefillOmen)
#   --------------------------------------------------------------------------------
#   Условие срабатывания (только **используемые** области, см. CraftExaltationAreas):
#   - если RemainingGreaterExaltationUses == 0 (Greater всегда используется), **или**
#   - если в CraftExaltationAreas есть Sinistral и RemainingSinistralExaltationUses == 0, **или**
#   - если в CraftExaltationAreas есть Dextral и RemainingDextralExaltationUses == 0
#
#   ВАЖНО про множественное срабатывание:
#   - после актуализации Remaining* выполняем один (и только один) проход HandleRemaining...;
#   - затем сразу обрываем текущую ветку крафта (без повторных проверок и без второго вызова RefillOmen),
#     чтобы AddTwoAffixes (когда сразу несколько Remaining* стали 0) не вызвал пополнение несколько раз.
#
#   Актуализация Remaining* внутри ветки (AddTwoAffixes / AddOneAffix):
#   - как только **любой используемый** счётчик Remaining* стал **0**, **немедленно** выходим из ветки крафта
#     и переходим к **HandleRemainingExaltationUsesDepletion** (цепочка с **RefillOmen**);
#   - **не** доводим до конца актуализацию остальных Remaining* в том же шаге ветки
#     (порядок уменьшения при нескольких расходах за один логический шаг — на усмотрение кода).
#
#   RefillOmen — пополнение омнов (фиксированный порядок действий)
#   -------------------------------------------------------------
#   Области экрана (имена как в настройках/коде):
#   - _ritualInventoryRegion
#   - _omenSinistralStashRegion
#   - _omenDextralStashRegion
#   - _omenGreaterStashRegion
#   - _currencyInventoryRegion
#
#   1) Общий шаг (всегда, до ветвления по областям):
#        - **ЛКМ** по области **_ritualInventoryRegion** (открыть/сфокусировать ritual inventory).
#
#   Далее — в зависимости от **CraftExaltationAreas** (те же ветки, что prefix_only / suffix_only / mixed):
#
#   1.1) Если в крафте участвует только **Sinistral** (prefix_only по правилу областей):
#        - **35 раз**: **Ctrl+ЛКМ** по области **_omenSinistralStashRegion**
#        - **25 раз**: **Ctrl+ЛКМ** по области **_omenGreaterStashRegion**
#
#   1.2) Если в крафте участвует только **Dextral** (suffix_only):
#        - **35 раз**: **Ctrl+ЛКМ** по области **_omenDextralStashRegion**
#        - **25 раз**: **Ctrl+ЛКМ** по области **_omenGreaterStashRegion**
#
#   1.3) Если **mixed** (только Greater в наборах областей):
#        - **1 раз**: **Ctrl+ПКМ** по области **_omenGreaterStashRegion**
#
#   2) После выполнения **одной** из веток 1.1 / 1.2 / 1.3 (согласно CraftExaltationAreas):
#        - **ЛКМ** по области **_currencyInventoryRegion**.
#      Между шагом 2 и шагом 3 **нет** второго ЛКМ по **_ritualInventoryRegion** — сразу Refresh.
#
#   3) **RefreshRemainingExaltationOmenStacks** — обновить **Remaining*** по всем нужным сеткам
#      (как в обычном описании функции).
#
#   4) Обойти **все ячейки** сеток омнов в **используемых** в крафте областях (по **CraftExaltationAreas**:
#      область **Greater** всегда; **Sinistral** / **Dextral** — только если входят в набор).
#      В **каждой** такой ячейке выполнить **ПКМ** (одиночный клик по ячейке сетки, без Ctrl).

#       - как только нашли первую не пустую ячейку с нужным omen:
#           Ctrl+ПКМ по этой ячейке (убрать оставшиеся Greater omens из stash) и выйти из поиска

#   A2) RemainingGreaterExaltationUses := 0
#       и считаем, что все области пусты для целей продолжения ветки:
#       RemainingSinistralExaltationUses := 0

#   A3) Вызвать RefillOmen
#   A4) После завершения RefillOmen (шаги 1–4, см. блок RefillOmen: в т.ч. Refresh и ПКМ по ячейкам):

#       - если CraftExaltationAreas включает Sinistral — работаем с Sinistral
#       - иначе если CraftExaltationAreas включает Dextral — работаем с Dextral

#       - Ctrl+C на ячейке

#      - обрывать текущую ветку крафта (без дополнительных действий с оменами).

# Decision flow
# -------------
#
# Вход в крафт по ячейке предмета: сначала блок Init craft (переменные + Precheck).
# Далее цикл: после любой завершённой ветки (без успеха по условию) возврат на **1.1**,
# не в Init craft. Rarity за время крафта не меняется — повторный Precheck в цикле не выполняем.
#
#   +-------------------------------------------------------------+
#   | START: ExaltationCraftServiceFracturedSide for one item cell |
#   +-------------------------------------------------------------+
#                     |
#                     v
#   +-------------------------------------------------------------+
#   | Init craft — один раз при входе в крафт по этой ячейке       |
#   | - PreviousCraftBranch := A (см. п. 4 «Переменные»);          |
#   | - CraftExaltationAreas ← resolveCraftExaltationAreas;        |
#   | - EnsureInitialExaltationOmenSupply (см. раздел выше):     |
#   |   Refresh → при 0 RefillOmen (внутри — Refresh + ПКМ);      |
#   |   проверка остатков; если 0 — STOP, ошибка пополнения;      |
#   | - 1) Precheck rarity: item MUST be Rare                     |
#   | Инвариант: в ходе крафта Rarity предмета не меняется.       |
#   +-------------------------------------------------------------+
#          | ok                              | not Rare
#          v                                 v
#          |                          +---------------------------+
#          |                          | STOP (fail / skip cell)   |
#          |                          +---------------------------+
#          v
#   +-------------------------------------------------------------+
#   | 1.1) Determine capacity (defaults): P_max=3, S_max=3         |
#   |      (exceptions list — later)                              |
#   |      [вход LOOP после завершения ветки — см. низ схемы]      |
#   +-------------------------------------------------------------+
#                     |
#                     v
#   +-------------------------------------------------------------+
#   | 1.2) Count current affixes: P_cur / S_cur                    |
#   +-------------------------------------------------------------+
#                     |
#                     v
#   +-------------------------------------------------------------+
#   | 2) Evaluate craft condition vs current item:                 |
#   |    - compute match_count                                     |
#   |    - derive "condition complete?"                            |
#   |    - derive prefix_only / suffix_only / mixed (по плану)     |
#   +-------------------------------------------------------------+
#                     |
#          +----------+-----------------------------+
#          |                                        |
#          v                                        v
#   +---------------------------+           +---------------------------+
#   | 7) condition complete?    |           | condition NOT complete     |
#   | -> STOP (success)         |           +---------------------------+
#   +---------------------------+                        |
#                                                        v
#                             +--------------------------+---------------------------+
#                             | Branching by match_count                               |
#                             +--------------------------+---------------------------+
#                                        | match_count == 0                | match_count > 0
#                                        v                                 v
#                 +----------------------+--------------------+     +------------------------------+
#                 | A) No matches (0)                           |     | B) Have matches (>0)         |
#                 +----------------------+--------------------+     +------------------------------+
#                            |                                            |
#                            v                                            v
#           +----------------+---------------------+      +---------------+----------------------+
#           | A1) can_add_2 ? (есть место для 2)   |      | B1) condition not complete (given) |
#           +----------------+---------------------+      +---------------+----------------------+
#                   | yes                    | no                         |
#                   v                        v                            v
#   +---------------+------------------+   +---------------------------+  +-----------------------------+
#   | CALL AddTwoAffixes                 |   | A5) Apply Annulment x2    |  | B2) can_add_1 ? (есть место |
#   | (см. описание выше)                |   |     (ветка завершена)     |  |     для 1 аффикса)          |
#   +---------------+------------------+   +---------------------------+  +-----------------------------+
#                   |                                                          | yes                 | no
#                   v                                                          v                     v
#          +---------------------------+                         +--------------+--------------+   +---------------------------+
#          | (ветка завершена)         |                         | CALL AddOneAffix                |   | B5) Apply Annulment x1    |
#          +---------------------------+                         | (см. описание выше)             |   |     (ветка завершена)     |
#                                                                +--------------+--------------+   +---------------------------+
#                                                                                |
#                                                                                v
#                                                                +------------------------------+
#                                                                | (ветка завершена)            |
#                                                                +------------------------------+
#                                                                                |
#   LOOP: перейти к шагу **1.1** — к блоку «1.1) Determine capacity…» в начале этой схемы
#   (сразу после Init craft). Повторно Init craft и 1) Precheck не выполняются.
#
#
# Global termination rules (apply to all branches)
# ------------------------------------------------
# - If at any point we cannot activate required omen because there are no more omens
#   in the configured area => STOP craft loop.
# - If craft condition becomes complete at any point => STOP craft loop (success).
#
