# Market Ratio и биржевой стакан (PoE2)

## Поведение в игре

1. В окне **Market Ratio** при наведении на панель с заголовком «MARKET RATIO» и текущим курсом (например `1 : 216`) игра показывает **стакан**: до **5** строк **Available Trades** (лучшие предложения на продажу) и до **5** строк **Competing Trades** (лучшие заявки на покупку).
2. Чтобы стакан не перекрывался курсором: навести мышь на панель, зажать **Alt**, затем сдвинуть курсор **вправо** (в GameHelper по умолчанию **200 px**).

## Примеры интерфейса (скриншоты)

См. папку [examples/](examples/):

- `poe2_market_ratio_panel_example.png` — панель **MARKET RATIO** с курсом (область наведения).
- `poe2_market_order_book_example.png` — полное окно стакана (**AVAILABLE TRADES** / **COMPETING TRADES**).

Если файлов нет, скопируйте свои скриншоты с этими именами в `docs/examples/` для справки.

## Настройка в GameHelper

На вкладке **«Сбор информации»**, блок Market Ratio:

| Область | Назначение |
|--------|------------|
| Наведение для стакана | Прямоугольник по панели Market Ratio (как на первом примере). |
| OCR стакана | Большой прямоугольник по **всему** всплывающему окну со стаканом при удержании Alt (как на втором примере). |
| Сдвиг после Alt, px | Смещение курсора вправо (по умолчанию 200). |

После успешного выставления пары **Exalted → Divine** сценарий: OCR курса/золота → запись в `Log/currency_exchange_rates.csv` → наведение → **Alt** → сдвиг → OCR стакана.

## Расчёт спреда (арбитраж) — отдельная функция

Логика вынесена в **чистый** класс без UI:

- `GameHelper.Services.CurrencyPairArbitrageCalculator.TryComputeAskMinusBidSpread(bestAvailableRatioRight, bestCompetingRatioRight)`

По умолчанию для пары строк вида **1 : R** берутся **правые части** первой строки **Available Trades** и первой строки **Competing Trades** (после эвристического разбора `OrderBookOcrParser`). Возвращается **askRight − bidRight** (или `null`, если не удалось извлечь оба числа).

**Интерпретация** зависит от направления сделки и экономики пары; функция только считает разницу в тех же единицах, что и на экране. Для других пар валют вызывайте ту же функцию с числами, полученными своим парсером или вручную.

## Разбор OCR

`OrderBookOcrParser` ищет секции по подстрокам **AVAILABLE**/**TRADE** и **COMPETING**/**TRADE**, затем строки с шаблоном курса `число : число` (в т.ч. десятичная правая часть). Надёжность зависит от качества Windows OCR; при сбоях смотрите сырой файл в `order_book_raw/`.
