# Changelog

## v0.2.0 — 2026-06-06

### Окно лога крафта
- Новое окно `CraftLogWindow`: открывается кнопкой или горячей клавишей, показывает содержимое лога текущего или последнего крафта
- Автообновление раз в секунду во время активного крафта (`.tmp`-файл); при завершении крафта окно находит итоговый `.txt` и показывает финальный результат включая строку «Завершение крафта»
- Автопрокрутка к последней строке; если пользователь прокрутил вверх вручную — позиция не сбрасывается
- Содержимое сохраняется при закрытии `.tmp`-файла — окно не теряет лог после завершения крафта
- `Topmost = true` — окно видно поверх игры; `ShowActivated = false` — не крадёт фокус у PoE2

### Глобальные горячие клавиши
- Горячая клавиша для сворачивания/разворачивания окна в трей (настраивается)
- Горячая клавиша для открытия/закрытия окна лога крафта (настраивается)
- Горячая клавиша для старта/стопа крафта (настраивается)
- **Исправление**: каждая пользовательская горячая клавиша регистрируется в 8 вариантах модификаторов (`±Alt ±Ctrl ±Shift`), чтобы срабатывать пока крафт-сервис программно удерживает Shift/Ctrl через `keybd_event`

### Запоминание позиции и размера окон
- Все окна (`MainWindow`, `CraftLogWindow`, `CraftConditionWindow`, `RegionPickerWindow`, `ItemParsingWindow`) сохраняют позицию и размер в `window_geometry.json`
- При свёрнутом окне сохраняются координаты до сворачивания (`RestoreBounds`), а не `-32000`
- Позиция восстанавливается только если окно попадает на видимую область экрана

### Диалоговые окна поверх главного окна
- Все `MessageBox.Show` получили `this` как owner — ошибки больше не уходят под основное окно

### ARCH-6: единая модель `CraftResult`
- Новый `Services/CraftResult.cs`: `record CraftResult(ChaosCraftResult StopReason, int Attempts, string? FinalItem)`
- Все три крафт-сервиса (`ChaosCraftService`, `AugAnnulCraftService`, `ExaltationCraftServiceFracturedSide`) используют `CraftResult` вместо кортежа `(ChaosCraftResult, int)`
- Фабричные методы: `CraftResult.Found(...)`, `.Stopped(...)`, `.LimitReached(...)`, `.Empty(...)`, `.Failed(...)`

### QA-1: пустые catch-блоки
- `SettingsStore.Load` и `SettingsStore.Save` логируют исключения через `SessionLogger` вместо молчаливого проглатывания

### Библиотека аффиксов: диапазоны по классу предмета
- Исправлена ошибка: аффикс с одним именем и тиром мог иметь разные диапазоны значений на разных типах предметов (например, «of Frostbite T1» на Wands и Staves), но сохранялся в одну запись с одним набором диапазонов
- Теперь ключ записи включает класс предмета; каждый класс получает собственную запись с корректными диапазонами
- Автоматическая миграция при загрузке: записи с несколькими классами разбиваются на отдельные записи (по одной на класс)
