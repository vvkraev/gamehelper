# Changelog

## v0.5.0 — 2026-06-22

### Режим шансинга (Chancing)
- `ChancingService`: применяет Orb of Chance к таблетам в двух режимах — по одному (Single) и по сетке (Grid)
- Режим с Omen of the Ancients: гарантирует уникальный результат, поддерживает cross-base
- Статистика сессии: количество Unique / Destroyed, ROI, EV/попытку, разбивка по именам
- Экономика: цены ресурсов (OoC, Omen, таблет) и уникальных результатов в Exalted; автоматический расчёт прибыли

### Вкладка Справочник (Reference)
- Статистика распределения аффиксов накапливается из логов Chaos-крафта
- `ReferenceStatsService` + `AffixStatsScanner`: читают лог-файлы и строят частотные таблицы
- `AffixStatsData`: модели данных статистики с накоплением между сессиями
- `docs/stats/` — хранилище накопленной статистики по форме JSON

### Socketable: под-вкладки в Networth
- Runes, Kulguuran Runes, Soul Cores, Idols, Augments — отдельные под-разделы
- Сканирование со своими областями; цены подтягиваются из poe.ninja

## v0.4.0 — 2026-06-12

### Networth
- `NetworthService`: сканирует вкладки стэша по заданным областям, суммирует стоимость через `PoeNinjaPriceService`
- OCR-навигация к стэшу (Windows OCR), клик на нужную вкладку и под-вкладку (`NavTabRect` + `TabRect`)
- `NetworthSnapshotStore`: сохраняет результат в JSON, загружает последний снэпшот при старте

### Repricing
- `RepricingService`: переоценка выставленных товаров в асинхронном трейде
- Проходит по вкладкам, считывает текущую цену из буфера (`Note: ~b/o`), снижает по ступенчатому алгоритму
- Детектирует открытие диалога цены через Ctrl+C после ПКМ, верифицирует результат повторным чтением
- Повторный цикл по таймеру с настраиваемым интервалом
- `RepricingTabSettingsWindow`: UI для настройки вкладок, ячеек и шагов снижения

### Delirium
- Панель настройки областей Delirium: инвентарь + области предметов
- 26 Liquid Orbs добавлены в `StackableItemRegistry` через `stackable_item_types.json`

### Stackable Item Registry
- `StackableItemRegistry`: реестр всех стакуемых предметов (рунные камни, катализаторы, орбы, Delirium)
- `stackable_item_types.json` — источник данных, копируется вместе с проектом

## v0.3.0 — 2026-06-06

### Перековка (Reforge)
- `ReforgeService`: полный цикл перековки — сканирование инвентаря, пакетная загрузка в станок (≤3 стака), кнопка Reforge, чтение результата с retry, извлечение остатков
- `AutoReforgeService`: вариант с auto-scan + auto-batch без ручного ввода
- `ReforgeState`: сохраняемое состояние между сессиями

### Каскад аффиксов
- `CraftAffixCascadeHelper`: улучшенный каскадный подбор аффиксов при Aug+Annul крафте
- `AffixCraftPatternBuilder`: построение паттернов для поиска в библиотеке

### Статистика катализаторов
- `CatalystReforgeStatsScanner`: сбор статистики по выходам перековки
- `catalyst_reforge_stats.json`: накопленные данные

### Управление оменами (AutoRefillOmen)
- `OmenActivationService`: рефил стаков оменов (Refresh + ПКМ), обработка остатков, режим Greater refresh
- `ExaltationCraftServiceFracturedSide`: интеграция авто-рефила в цикл Exalt-крафта

## v0.2.0 — 2026-06-01

### Окно лога крафта
- `CraftLogWindow`: открывается кнопкой или горячей клавишей, автообновление из `.tmp`-файла во время крафта
- Автопрокрутка; `Topmost = true`, `ShowActivated = false` — видно поверх игры без кражи фокуса

### Глобальные горячие клавиши
- Свернуть/развернуть в трей, открыть лог крафта, старт/стоп крафта
- Каждая клавиша регистрируется в 8 вариантах модификаторов — срабатывает при удержании Shift/Ctrl сервисом

### Запоминание позиции окон
- Все окна сохраняют положение и размер в `window_geometry.json` через `WindowGeometryStore`

### ARCH-6: единая модель CraftResult
- `CraftResult`: record вместо кортежа; фабричные методы `Found`, `Stopped`, `LimitReached`, `Empty`, `Failed`
- Все три крафт-сервиса используют единую модель

### Библиотека аффиксов: диапазоны по классу предмета
- Ключ записи теперь включает класс предмета — исправлены расхождения диапазонов между Wands и Staves
